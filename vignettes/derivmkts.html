Option Pricing Functions to Accompany Derivatives Markets

Robert McDonald

2015-10-02

Introduction

This provides documentation for the R package derivmkts, which is meant as a companion to Derivatives Markets by Robert McDonald. There are of course other option pricing packages, but this package uses function names corresponding to the usage in the book.

Basic Pricing Functions

Black-Scholes

The bscall, bsput, and bsopt provide basic pricing of European calls and puts. There are also binary payoffs: cash-or-nothing and asset-or-nothing options. All functions are vectorized, and bsopt provides option greeks. Here are some examples:

s <- 100; k <- 100; r <- 0.08; v <- 0.30; tt <- 0.25; d <- 0
bscall(s, k, v, r, tt, d)
## [1] 6.961842
bsput(s, k, v, r, tt, d)
## [1] 4.981709
bsput(s, c(95, 100, 105), v, r, tt, d)
## [1] 2.970771 4.981709 7.637441
bsopt(s, c(95, 100, 105), v, r, tt, d)$Call

Asian Options

There are pricing functions for a large number of Asian options:

  • down-and-in and down-and-out barrier binary options
  • up-and-in and up-and-out barrier binary options
  • more standard down- and up- calls and puts, constructed using the barrier binary options

Naming for the barrier options generally follows the convention

[u|d][i|o][call|put]

which means that the option is “up” or “down”, “in” or “out”, and a call or put.1 For binary options, we add the underlying asset:

[asset|cash][u|d][i|o][call|put]

uicall(s, c(95, 100, 105), v, r, tt, d, 102)
## [1] 9.826595 6.667587 3.508579

  1. The naming convention here differs from that in Derivatives Markets, in which names are callupin, callupout, etc. For consistency,, these functions have both names.↩
